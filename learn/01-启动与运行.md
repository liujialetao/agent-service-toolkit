## `.env` 配置模板

文件位置：**项目根目录 `.env`**（已在 `.gitignore`，勿提交 Git）

```env
USE_FAKE_MODEL=false

# 阿里云百炼 OpenAI 兼容接口
COMPATIBLE_MODEL=qwen-max          # 或 qwen3.6-flash 等
COMPATIBLE_API_KEY=<百炼控制台申请>
COMPATIBLE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DEFAULT_MODEL=openai-compatible

# 本地 SQLite，无需 PostgreSQL
DATABASE_TYPE=sqlite
SQLITE_DB_PATH=checkpoints.db

# 开发模式（热重载）
MODE=dev
LOG_LEVEL=INFO

# 后端监听地址（服务绑定用）
HOST=0.0.0.0
PORT=8080

# ⚠️ 重要：Streamlit 连接后端必须用 127.0.0.1
AGENT_URL=http://127.0.0.1:8080
```

### 关键变量说明

| 变量 | 作用 |
|------|------|
| `COMPATIBLE_MODEL` | 百炼实际模型名（如 `qwen-max`） |
| `COMPATIBLE_API_KEY` | 百炼 API Key |
| `DEFAULT_MODEL` | 固定为 `openai-compatible`（项目内部类型，不是百炼模型名） |
| `HOST` | 后端**监听**地址，保持 `0.0.0.0` 即可 |
| `AGENT_URL` | 前端**连接**后端的地址，必须是 `http://127.0.0.1:8080` |

Key 申请：https://dashscope.console.aliyun.com/

## 启动（推荐：两个终端，前台运行）

在 **Cursor** 中按 `` Ctrl+` `` 打开终端，开两个标签页。

### 终端 1：FastAPI 后端

```bash
cd /Users/liujia/code/learn/rag/githubDemos/agent-service-toolkit/src
../.venv/bin/python run_service.py
```

正常输出示例：

```text
INFO:     Uvicorn running on http://0.0.0.0:8080
INFO:     Application startup complete.
```

**请求日志在这个终端实时输出**（HTTP 请求、Agent 报错等）。

### 终端 2：Streamlit 前端

```bash
cd /Users/liujia/code/learn/rag/githubDemos/agent-service-toolkit/src
../.venv/bin/streamlit run streamlit_app.py --server.headless true
```

正常输出示例：

```text
Local URL: http://localhost:8501
```

> 项目已包含 `.streamlit/credentials.toml`，可跳过 Streamlit 首次运行的邮箱提示。

若 **8501 已被占用**，Streamlit 会自动改用 **8502**（或下一个可用端口），以终端里 `Local URL:` 那一行为准。

### 访问地址

| 服务 | URL |
|------|-----|
| 聊天 UI | http://localhost:8501（8501 占用时见终端，常见为 8502） |
| API 文档 | http://localhost:8080/docs |
| Agent / 模型列表 | http://localhost:8080/info |
| 健康检查 | http://localhost:8080/health |

## 验证服务是否正常

两个服务需**同时**在跑：后端 `run_service.py`（8080）+ 前端 `streamlit_app.py`（8501 或 8502）。

### 1. 查看进程是否在运行（最直接）

```bash
pgrep -fl "run_service.py|streamlit_app.py"
```

| 结果 | 含义 |
|------|------|
| 有输出（含 PID、命令行） | 对应进程仍在运行 |
| 无输出 | 进程已停止 |

示例输出（两个都在跑时）：

```text
24301 ../.venv/bin/python run_service.py
24300 .../streamlit run streamlit_app.py --server.headless true
```

等价写法：

```bash
ps aux | grep -E "run_service|streamlit_app" | grep -v grep
```

### 2. 查看端口是否在监听

```bash
lsof -nP -iTCP:8080 -sTCP:LISTEN    # FastAPI 后端
lsof -nP -iTCP:8501 -sTCP:LISTEN    # Streamlit 默认端口
lsof -nP -iTCP:8502 -sTCP:LISTEN    # 8501 被占用时的常见备选端口
```

| 端口 | 服务 |
|------|------|
| 8080 | FastAPI 后端 |
| 8501 / 8502 | Streamlit 前端（以终端 `Local URL` 为准） |

### 3. 查看 HTTP 是否可访问

```bash
curl -s http://127.0.0.1:8080/health
# 正常应返回：{"status":"ok"}

curl -s -o /dev/null -w "8501 HTTP %{http_code}\n" http://127.0.0.1:8501
curl -s -o /dev/null -w "8502 HTTP %{http_code}\n" http://127.0.0.1:8502
# 正常应返回：HTTP 200
```

### 4. 一键检查（推荐）

复制整段到终端，一次看完进程、端口、HTTP 状态：

```bash
echo "=== 进程 ==="
pgrep -fl "run_service.py|streamlit_app.py" || echo "无相关进程"

echo "=== 端口 ==="
lsof -nP -iTCP:8080 -sTCP:LISTEN || echo "8080 未监听"
lsof -nP -iTCP:8501 -sTCP:LISTEN || echo "8501 未监听"
lsof -nP -iTCP:8502 -sTCP:LISTEN || echo "8502 未监听"

echo "=== HTTP ==="
curl -s http://127.0.0.1:8080/health && echo
curl -s -o /dev/null -w "8501 HTTP %{http_code}\n" http://127.0.0.1:8501
curl -s -o /dev/null -w "8502 HTTP %{http_code}\n" http://127.0.0.1:8502
```

### 查看 Agent 和模型

```bash
curl -s http://127.0.0.1:8080/info | python3 -m json.tool
```

`models` 中应有 `openai-compatible`。

## 常见问题

### 1. Streamlit 首次启动卡在 `Email:`

Streamlit 第一次运行的欢迎提示，**直接按 Enter 留空**即可。

项目已有 `.streamlit/credentials.toml` 可避免此问题。

### 2. 前端报 502：`Error connecting to agent service at http://0.0.0.0:8080`

**原因**：macOS 上 Python `httpx` 访问 `http://0.0.0.0:8080` 会返回 502（`curl` 可能正常）。

**解决**：`.env` 中添加：

```env
AGENT_URL=http://127.0.0.1:8080
```

然后重启 Streamlit（建议两个服务都重启）。

### 3. 聊天时报 `Internal server error`

看**后端终端**的 `Error in message generator:` 行。

常见原因之一：百炼内容安全审核拦截，例如：

```text
DataInspectionFailed: Input text data may contain inappropriate content
```

多见于 `research-assistant` 调用 WebSearch 后，搜索结果触发审核。可换 `chatbot` 验证 LLM 通路，或换模型/调整问题。

### 4. 后台启动 vs 前台启动

Agent / Cursor 后台启动的进程没有可见终端，**看不到实时日志**。

学习阶段建议始终在**自己的终端前台**启动，便于观察请求和报错。
